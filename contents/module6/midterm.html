Part 5  ·  AI-Critical Thinking5 pts

Suggested time: 20 minutes

Context: The exchange below shows a student asking an AI tool for help during a homework session. You are not being asked whether using AI was appropriate. You are being asked to think critically about the output the AI produced, verify it against the actual files in your workspace, and explain what you find.

Student prompt:

"Write an awk command to show me all lab results where the glucose value is above 100. The file is called lab_results.tsv and has these columns: patient_id, visit_date, test_name, result_value, unit, status."

AI response:

Sure! Here is a command that filters glucose results above 100:

awk -F ',' 'NR==1 || ($3=="Glucose" && $4+0 > 100) {print}' lab_results.tsv

This keeps the header (NR==1) and prints rows where field 3 is Glucose and field 4 is greater than 100.

Write your answers as echo statements appended to outputs/ai_analysis.txt .

  1. (2 pts) Identify the error: Run the AI's command exactly as given on data/lab_results.tsv . What does the output look like? Is it correct? State precisely what is wrong and why.
  2. (2 pts) Fix and verify: Write the corrected command. Append the corrected command as an echo line to outputs/ai_analysis.txt , then save the actual output to outputs/ai_corrected_output.tsv and verify it contains the expected rows.
  3. (1 pt) Lesson: In one sentence, explain what step you would always take before trusting an AI-generated awk or sed command applied to a new file.
